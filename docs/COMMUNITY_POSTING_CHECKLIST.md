# Monthly Community Posting Checklist

Use this checklist for each release cycle to keep distribution consistent.

## Inputs

- release version
- changelog summary
- one short CLI demo snippet
- one troubleshooting link

## Channel Checklist

### X

- [ ] Post release summary in <= 280 chars
- [ ] Include one concrete result (QA/coverage/reliability)
- [ ] Add repository link

### Reddit

- [ ] Post technical context (problem, approach, trade-offs)
- [ ] Include command examples
- [ ] Respond to first 3 comments within 24h

### Dev.to / Blog

- [ ] Publish longer writeup with architecture section
- [ ] Include screenshots or output snippets
- [ ] Link to troubleshooting and contributing docs

## Metrics to Record

- stars gained in 7 days
- new issues opened
- PRs from new contributors
- npm downloads (weekly)

## Retrospective

- best performing channel
- key objections/questions from users
- one concrete doc or product change to ship next
